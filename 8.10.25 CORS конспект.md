**CORS (Cross-Origin Resource Sharing)** — это механизм безопасности браузеров, который контролирует доступ к ресурсам на веб-страницах из разных источников.

**Источник (origin)** определяется тремя параметрами:
- **Протокол** (http/https)
- **Порт** (80/443)
- **Домен** (`example.com`)

**Типы запросов:**
- **Запись** (формы, ссылки) — разрешена по умолчанию
- **Вставка** (img, script, video) — разрешена по умолчанию
- **Чтение** (AJAX/fetch) — заблокирована по умолчанию

**Предварительные запросы отправляются браузером перед сложными запросами, когда:**
- Метод не GET/POST/HEAD
- Используются нестандартные заголовки
- Content-Type отличается от стандартных значений

**Основные заголовки CORS:**
- `Access-Control-Allow-Origin` - разрешенные домены
- `Access-Control-Allow-Methods` - разрешенные методы
- `Access-Control-Allow-Headers` - разрешенные заголовки
- `Access-Control-Allow-Credentials` - разрешение кук

**Рекомендации**:
1. **Для публичных API** - использовать `Access-Control-Allow-Origin: *`
2. **Для защищенных ресурсов** - указывать конкретные домены
3. **С куками** - избегать комбинации `*` с credentials
4. **В разработке** - использовать плагины для отключения CORS или настроить прокси
#### Зачем нужен CORS?
Чтобы защитить от:
- Межсайтовых атак (CSRF)
- Утечки конфиденциальных данных
- Неавторизованного доступа к API

**CORS** — это важный механизм безопасности, который нужно правильно понимать и настраивать.
