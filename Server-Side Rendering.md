# Server-Side Rendering (SSR)

  

[[Архитектура и серверная разработка]]

  
  

Рендеринг HTML на сервере

  

### SSR (Server-Side)

  

- HTML генерируется на сервере

- Лучше для SEO

- Быстрая первая загрузка

- Больше нагрузка на сервер

  

### CSR (Client-Side)

  

- HTML генерируется в браузере

- Интерактивнее

- Медленная первая загрузка

- Нагрузка на клиента

  

**Опасность SSTI (Server-Side Template Injection):**

  

Если пользовательский ввод попадает напрямую в шаблон, злоумышленник может выполнить произвольный код на сервере!

  

### Правила безопасного SSR

  

- Никогда не использовать render_template_string() с пользовательским вводом

- Включить автоэкранирование в шаблонизаторе

- Использовать Content Security Policy (CSP)

- Валидировать все данные перед рендерингом

- Разделять логику и представление (MVC паттерн)

  

Что такое CSRF?

Атакующий заставляет жертву выполнить нежелательное действие на доверенном сайте, где жертва аутентифицирована.

### Последствия CSRF

  

- Несанкционированные платежи

- Изменение пароля/email

- Удаление аккаунта

- Публикация контента от имени жертвы
